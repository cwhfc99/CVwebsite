Classroom Reward Web-Game (Dissertation)

My final year university dissertation is largest and most complex project I have worked on. I opted to undertake a software engineering project and was tasked with researching, designing, implementing, testing, evaluating and reporting on, an innovating product. I chose to create a web-based system aimed at the education. The idea was to use a game to motivate students instead of educate. I create an integrated database-driven RPG game and digitalised classroom reward system, whereby a teacher could reward students with in-game currency for real-life classroom activities, in an attempt to facilitate extrinsic motivation. The product created is a prototype which lacks suitable graphics and scalability to be a commercially viable product, but serves as a proof of concept and a demonstration of my technical ability. The system is written predominately in PHP/JavaScript/HTML/CSS and is driven by a PhpMyAdmin database. It has not been created using any web templates or frameworks, but does make use of open-source PHP libraries.

Abstract from accompanying project report.
This report documents the process of developing a product that pairs a digital classroom reward system with a computer game. It outlines the Research, Design, Development and Evaluation phases completed in the creation of a web-based prototype. It also includes all the design and testing documentation produced throughout this process. The product incorporates a database- driven RPG game, with teachers able to reward students with in-game currency, with the intention of providing extrinsic motivation within an educational context. An understanding of how motivation is created, from a psychological perspective, is explained. Many technical software development decisions are discussed in this report, with notable relevance to PHP and database design, along with theoretical ideas/features discovered from relevant literature and analysis of existing systems. User evaluation was conducted to assist with the evaluation of the project and this is documented. The report concludes by offering the findings from the project and suggestions for future development projects, within the field of E-learning, with pertinence to an innovative game-based approach to the creation of educational/motivation software and the design of an intuitive, easy-to-use reward allocation system.

The full accompanying report is available here and I would encourage you to read it, in order to contextualise the finished product and because it is an excellent demonstration of my written communication skills.

The product created is available here. This is hosted on a university server and may not be available forever. This is by no means a finished product and should be considered a prototype.

To view the source code for this project please visit this Github repository. Please note, this code has been marginally edited from from that used in the actual product for security reasons and will not work should it be hosted else where.

CV Website

I have chosen to include this website as part of my portfolio as a demonstration of web development capabilities. From a design point of view, I wanted this site to feel like a continuation of my CV. The theme is the same as my CV and I have chosen to keep all the content on the same page so that it reads like a more detailed version of my CV. This is designed like this to make it as easy as possible for the user to find the information they want. Items that are not as important are initially hidden from the user, so they are not completely swamped with information.

From a technical point of view, the site is written in HTML/CSS/JavaScript. I'm aware I could have used a web template or framework, but I was keen to demonstrate my problem solving skills and created all the scrolling animations using vanilla JavaScript. I consider this static website to be rather basic, but it serves it's purpose well.
